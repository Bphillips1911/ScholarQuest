PHASE C - FIX API ENDPOINTS & TENANT SCOPING

C1) Fix /api/educap/standards endpoint:
CURRENT PROBLEM: Returns empty array in production
ROOT CAUSE: Query filters by user.schoolId which doesn't match seeded data schoolId
FIX: Modify query to return standards where:
  - schoolId matches user.schoolId OR
  - schoolId is NULL (global standards) OR
  - status = 'active'
CRITICAL: Do not change response format, only fix query logic

Example fix:
// BEFORE (broken in prod)
const standards = await db.query.standards.findMany({
  where: eq(standards.schoolId, user.schoolId)
});

// AFTER (works in prod)
const standards = await db.query.standards.findMany({
  where: and(
    grade ? eq(standards.grade, parseInt(grade)) : undefined,
    subject ? eq(standards.subject, subject) : undefined,
    or(
      isNull(standards.schoolId),
      eq(standards.schoolId, user.schoolId)
    )
  )
});
