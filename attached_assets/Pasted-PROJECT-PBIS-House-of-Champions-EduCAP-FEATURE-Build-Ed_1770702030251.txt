3) MOVEMENT MAP BAND LOGIC (EXACT)
========================
Movement Map uses a single “bandScore” per student per window and subject.

bandScore precedence:
1) If assessment score exists for that subject/window: use scaledPercent = score/maxScore * 100
2) Else fallback to masteryCoveragePercent (computed from standards mastery)
3) If neither exists: student is “No Data” bucket

Bands (default; store in DB so admin can adjust later):
- PRO (Proficient): bandScore >= 70
- ON (On Track): 50 <= bandScore < 70
- DEV (Developing): 30 <= bandScore < 50
- HR (High Risk): bandScore < 30
- ND (No Data): missing in either window (special handling)

Movement cell keys:
- fromBand ∈ {PRO, ON, DEV, HR, ND}
- toBand ∈ {PRO, ON, DEV, HR, ND}

Grid:
- 5x5 where ND row/col is included (but visually separated)
Counts shown in each cell.
Click cell -> returns roster list with fromBand/toBand, plus growth metrics.

Movement Growth (default growth metric):
- MovementDelta = bandIndex(toBand) - bandIndex(fromBand)
  where HR=0, DEV=1, ON=2, PRO=3, ND=null
- MovementDelta categories:
  - Accelerated: +2 or +3
  - Typical: +1
  - Flat: 0
  - Decline: -1 or less
- If ND in either window => “Incomplete” (excluded from median growth but counted in completion KPI)

Scaled Growth:
- scaledPercentTo - scaledPercentFrom (if both exist)

Mastery Growth:
- masteryCoverageTo - masteryCoverageFrom
(where masteryCoverage = % of standards in Proficient/Mastered for subject+grade)

4) PROJECTIONS (MODEL + explanationJSON)
========================
Compute per student a projected likelihood of being Proficient by Final window.

Projection inputs:
- available windows among Baseline/Midpoint/Final
- score trend (scaledPercent)
- mastery coverage trend
- completion + timeOnTask validity
- recency

Output:
- probabilityProficient (0..1)
- projectionBand ∈ {Likely Proficient, On Track, Needs Acceleration, High Risk}
- confidence ∈ {High, Medium, Low}

Store projection in projection_outputs with explanationJSON (required for transparency):

explanationJSON schema (exact):
{
  "modelVersion": "insightstack-proj-v1",
  "computedAt": "ISO-8601",
  "scope": { "studentId": "...", "subject": "ELA|Math", "grade": 6|7|8, "schoolId": "..." },
  "inputs": {
    "fromWindow": "Baseline|Midpoint",
    "toWindow": "Midpoint|Final",
    "availableWindows": ["Baseline","Midpoint","Final"],
    "scaledPercents": { "Baseline": 0-100|null, "Midpoint": 0-100|null, "Final": 0-100|null },
    "masteryCoverage": { "Baseline": 0-100|null, "Midpoint": 0-100|null, "Final": 0-100|null },
    "daysBetween": number|null,
    "timeOnTaskMinutes": { "Baseline": number|null, "Midpoint": number|null, "Final": number|null },
    "validityFlags": ["LOW_TIME","MISSING_WINDOW","ANOMALY_SWING", ...]
  },
  "outputs": {
    "probabilityProficient": 0-1,
    "projectionBand": "Likely Proficient|On Track|Needs Acceleration|High Risk",
    "confidence": "High|Medium|Low",
    "projectedFinalPercent": 0-100|null
  },
  "rationale": [
    "bullet string 1",
    "bullet string 2",
    "bullet string 3"
  ],
  "recommendedNextSteps": [
    { "type": "standard", "standardId": "...", "label": "...", "why": "..." },
    { "type": "grouping", "label": "Small group: Evidence & Reasoning", "why": "..." }
  ]
}

Projection logic (v1, explainable):
- If >=2 windows with scaledPercent:
  - velocity = (to-from)/daysBetween
  - projectedFinalPercent = currentPercent + velocity*daysUntilFinal
- Blend with masteryCoverage:
  - blendedFinal = 0.7*projectedFinalPercent + 0.3*projectedMasteryEquivalent
- Convert to probability:
  - prob = sigmoid((blendedFinal - 70)/10)
- Confidence:
  - High: >=2 windows + no major validity flags
  - Medium: >=2 windows + minor flags OR 1 window + strong mastery coverage
  - Low: missing windows or only mastery fallback
