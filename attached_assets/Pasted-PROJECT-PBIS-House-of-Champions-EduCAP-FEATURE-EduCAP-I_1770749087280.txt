3) MOVEMENT MAP BAND LOGIC (DEFINITIONS + COLORS)
We need two independent concepts:
(A) Band identity (PRO/ON/DEV/HR/ND) = student status per window
(B) Movement direction tint = improvement/flat/decline comparing From→To

Band rules:
- PRO = Proficient (meets/exceeds threshold)
- ON  = On track (below proficient but on trajectory)
- DEV = Developing (below on track)
- HR  = High risk (significantly below)
- ND  = No data (missing result in either window)

Band assignment must be deterministic, driven by the existing EduCAP scoring model:
- If your result model already stores a band/level, use it as source of truth.
- Else compute band from scaledScore/percentCorrect using per-grade cut scores stored in DB.

Movement Map grid:
- Rows = From band
- Cols = To band
- Cell value = count of students moving row→col
- Cell click opens Student List filtered to that exact movement pair.

Color rules:
- Band identity uses a color DOT/legend:
   PRO = Green
   ON  = Blue
   DEV = Orange
   HR  = Red/Pink
   ND  = Gray
- Movement direction uses cell background tint:
   Improvement (To band is higher than From band) = light green tint
   Flat (same band) = neutral light gray/white
   Decline (To band is lower) = light red tint
- Must render a bottom legend:
   Band Legend (dot+label for PRO/ON/DEV/HR/ND)
   Movement Legend (improve/flat/decline)

Band ordering (highest→lowest): PRO > ON > DEV > HR > ND
ND treated specially:
- If From=ND and To!=ND => treat as improvement tint
- If From!=ND and To=ND => treat as decline tint
- ND→ND => flat

4) PROJECTIONS: explanationJSON SCHEMA (TRANSPARENT + STORED)
Projections must store BOTH numeric outputs and a human-readable explanation payload.

Create/normalize table: projection_outputs
Columns:
- id, tenantId, schoolId, studentId, subject, gradeId
- fromWindowId, toWindowId (comparison used for model)
- projectedBand (PRO/ON/DEV/HR)
- probProficient (0..1)
- probOnTrack (0..1)
- projectedScaledScore (nullable)
- computedAt (timestamp)
- explanationJSON (jsonb, required)

explanationJSON schema (MUST MATCH EXACTLY):
{
  "modelVersion": "InsightStack.v1",
  "inputs": {
    "fromWindow": "Baseline|Midpoint|Final",
    "toWindow": "Baseline|Midpoint|Final",
    "scaledGrowth": number,
    "movementGrowth": { "fromBand": "PRO|ON|DEV|HR|ND", "toBand": "PRO|ON|DEV|HR|ND", "delta": number },
    "masteryCoverage": { "attemptedStandards": number, "masteredStandards": number, "coveragePct": number },
    "recencyDays": number
  },
  "drivers": [
    { "type": "positive|negative|neutral", "label": string, "detail": string, "weight": number }
  ],
  "assumptions": [
    { "label": string, "detail": string }
  ],
  "recommendations": [
    { "standardId": string, "standardLabel": string, "reason": string, "priority": "High|Medium|Low" }
  ],
  "confidence": { "score": number, "label": "High|Medium|Low" }
}

Projection computation rules (v1):
- probProficient = sigmoid( a*scaledGrowth + b*coveragePct + c*movementDelta + d )
- coveragePct = masteredStandards / max(1, attemptedStandards)
- movementDelta numeric mapping: PRO=3, ON=2, DEV=1, HR=0, ND=-1
- Confidence:
   High if coveragePct>=0.6 and recencyDays<=30
   Medium if coveragePct>=0.35
   Low otherwise
- Store all driver weights used so results are explainable.
